# Índice de services

Use este índice quando já souber qual produto precisa executar e quiser confirmar o nome exato do `service` antes de montar a chamada.

<Info>
Todas as consultas abaixo usam `POST /api/service-api`. O produto executado é definido pelo campo `service` no body.
</Info>

## Como pesquisar melhor

A busca funciona melhor quando o termo aparece como título, alias ou texto da página. Se não souber o alias exato, pesquise pelo tipo de documento, dado ou problema que quer resolver.

<CardGroup cols={2}>
  <Card title="Tenho um CPF" href="#pessoa-fisica">
    Pesquise por `cpf`, `receita`, `score`, `risco`, `telefone`, `email`, `ocr`, `face` ou `processos`.
  </Card>
  <Card title="Tenho um CNPJ" href="#pessoa-juridica">
    Pesquise por `cnpj`, `receita`, `risco de crédito`, `sócios`, `domínios`, `cartão CNPJ` ou `compliance`.
  </Card>
  <Card title="Tenho uma imagem" href="/guides/service-api/ocr-service-api">
    Pesquise por `OCR`, `CNH`, `RG`, `cartão CNPJ`, `comprovante de endereço`, `base64` ou `image1`.
  </Card>
  <Card title="Quero copiar payload" href="/api-reference/services-pessoa-fisica">
    Vá para o API Reference quando precisar de body, curl, response resumido e erro comum.
  </Card>
</CardGroup>

### Atalhos de busca

| Digite na busca | O que deve aparecer |
| --- | --- |
| `SERVICE_OCR` ou `OCR CNH` | OCR React para RG/CNH e guia de OCR. |
| `cartão CNPJ` | OCR de cartão CNPJ e payload com `SERVICE_OCR_CNPJ_CARD`. |
| `comprovante de endereço` | OCR de comprovante e diagnóstico de imagem. |
| `face index` | Busca facial por selfie na base de faces. |
| `risco de crédito` | Services de score, rating, risco e crédito PF/PJ. |
| `processos` | Consultas jurídicas, certidões e antecedentes. |
| `telefone` ou `email` | Validações e histórico de contato. |

<Warning>
  Antes de executar a chamada, confirme qual alias está liberado no produto
  do cliente. O campo `service` deve receber esse alias de chamada. Em alguns
  casos, ele é mais curto que o alias técnico do parceiro exibido no catálogo.
</Warning>

Na prática: se o alias configurado no produto for diferente do alias abaixo,
use o alias do produto no body. Isso evita erro de acesso ao serviço mesmo
quando o produto está ativo.
