# Customers

Os endpoints de customers permitem consultar clientes cadastrados e alterar o
status ativo de um ou mais registros. Use esta área quando sua integração
precisar verificar se um cliente já existe na base, recuperar o `tokenCustomer`
ou ativar/inativar registros.

## Endpoints disponíveis

| Operação | Endpoint | Uso |
| --- | --- | --- |
| Consultar cliente | `GET /api/customer?documentKey={documentKey}` | Busca clientes por CPF ou CNPJ |
| Alterar status de cliente | `POST /api/changeStatusOfCustomer` | Ativa ou inativa um ou mais registros |

## Consultar cliente

```http
GET /api/customer?documentKey={cpf-ou-cnpj}
```

Use o parâmetro `documentKey` para informar o CPF ou CNPJ do cliente.

Autorização:

```http
Authorization: Bearer {jwt_token}
```

Exemplo de resposta:

```json
[
  {
    "tokenCustomer": "c65449cd-5ce5-40b8-9aa4-0726b118f73e",
    "type": null,
    "documentKey": "c65449cd-5ce5-40b8-9aa4-0726b118f73e",
    "name": null,
    "addInformation": null,
    "active": false,
    "imageUrl": null
  }
]
```

## Alterar status de cliente

```http
POST /api/changeStatusOfCustomer
```

Autorização:

```http
Authorization: Bearer {jwt_token}
Content-Type: application/json
```

Exemplo de body:

```json
[
  {
    "documentKey": "c65449cd-5ce5-40b8-9aa4-0726b118f73e",
    "active": false
  }
]
```

| Campo | Descrição |
| --- | --- |
| `documentKey` | Identificador do cliente |
| `active` | Define se o cliente deve ficar ativo ou inativo |
