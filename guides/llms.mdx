# Arquivos para LLMs

A documentacao tambem publica arquivos em texto simples e JSON para uso por
LLMs, assistentes de codigo e ferramentas como ChatGPT, Claude, Cursor e
Windsurf.

Esses arquivos ajudam a IA a entender rapidamente os guias, endpoints, services,
ambientes, exemplos e cobertura atual da API idCerberus.

## Arquivos disponiveis

| Arquivo | Quando usar |
| --- | --- |
| [`/llms.txt`](/llms.txt) | Indice curto da documentacao, com links principais e visao geral. |
| [`/llms-small.txt`](/llms-small.txt) | Resumo operacional para integracoes: ambientes, autenticacao, `POST /api/service-api`, fluxos e services documentados. |
| [`/llms-full.txt`](/llms-full.txt) | Conteudo completo consolidado dos guias, API Reference, OpenAPI e mapeamento de cobertura. |
| [`/llms-api-reference.txt`](/llms-api-reference.txt) | Versao operacional do API Reference, com services e exemplos de curl prontos para consulta por IA. |
| [`/llms-services-coverage.txt`](/llms-services-coverage.txt) | Mapa separado do que ja esta documentado e do que foi encontrado no onboarding/backend. |
| [`/services-catalog.json`](/services-catalog.json) | Catalogo estruturado em JSON para automacoes, validadores e agentes. |
| [`/coverage-report.json`](/coverage-report.json) | Relatorio estruturado com totais, categorias, pendencias e divergencias de coverage. |
| [`/examples/*.curl`](/examples/auth.hml.curl) | Chamadas prontas para homologacao e producao. |

## Uso recomendado

Para tirar duvidas rapidas sobre a estrutura da documentacao, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms.txt
```

Para pedir ajuda de implementacao, geracao de `curl`, escolha de service ou
explicacao de fluxo, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-small.txt
```

Para gerar exemplos de request a partir dos services documentados, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-api-reference.txt
```

Para analises mais completas, revisao de cobertura ou perguntas que dependem de
todo o conteudo da documentacao, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-full.txt
```

Para comparar o que ja esta na documentacao com o que foi encontrado no
onboarding/backend, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-services-coverage.txt
```

Para automacoes que precisam de dados estruturados, use:

```txt
https://react-it.github.io/api-docs-idcerberus/services-catalog.json
https://react-it.github.io/api-docs-idcerberus/coverage-report.json
```

Para importar exemplos prontos em ferramentas ou scripts, use:

```txt
https://react-it.github.io/api-docs-idcerberus/examples/auth.hml.curl
https://react-it.github.io/api-docs-idcerberus/examples/service-api-cpf.hml.curl
```

## Exemplo de prompt

```txt
Use esta documentacao como fonte principal:
https://react-it.github.io/api-docs-idcerberus/llms-small.txt

Preciso integrar a API idCerberus para consultar dados de CPF.
Explique o fluxo, gere o curl de autenticacao e depois o curl do service correto.
Nao invente services que nao estejam na documentacao.
```

## Boas praticas para IA

- Use homologacao para exemplos e testes.
- Use producao somente quando o usuario pedir explicitamente.
- Nunca exponha `client`, `secret`, token JWT real, CPF real, CNPJ real ou imagem real.
- Se um service nao estiver no catalogo, confirme no backend/onboarding antes de documentar.
- Para consultas externas, prefira sempre o endpoint `POST /api/service-api` com o campo `service`.

## Atualizacao dos arquivos

Os arquivos sao gerados a partir dos guias locais, do OpenAPI e do mapeamento de
cobertura dos services. Para atualizar localmente:

```bash
node scripts/generate-llms.mjs
```

No Windows, se o PowerShell bloquear `npm.ps1`, use:

```powershell
npm.cmd run generate:llms
```

Para rodar a verificacao de texto e regenerar os arquivos:

```bash
npm run check
```

O export da documentacao tambem executa a geracao antes de publicar os arquivos.
