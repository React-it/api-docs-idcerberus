# Arquivos para LLMs

A documentação também publica arquivos em texto simples e JSON para uso por
LLMs, assistentes de código e ferramentas como ChatGPT, Claude, Cursor e
Windsurf.

Esses arquivos ajudam a IA a entender rapidamente os guias, endpoints, services,
ambientes, exemplos e cobertura atual da API idCerberus.

## Arquivos disponíveis

| Arquivo | Quando usar |
| --- | --- |
| [`/llms.txt`](/llms.txt) | Índice curto da documentação, com links principais e visão geral. |
| [`/llms-small.txt`](/llms-small.txt) | Resumo operacional para integrações: ambientes, autenticação, `POST /api/service-api`, fluxos e services documentados. |
| [`/llms-full.txt`](/llms-full.txt) | Conteúdo completo consolidado dos guias, API Reference, OpenAPI e mapeamento de cobertura. |
| [`/llms-api-reference.txt`](/llms-api-reference.txt) | Versão operacional do API Reference, com services e exemplos de curl prontos para consulta por IA. |
| [`/llms-services-coverage.txt`](/llms-services-coverage.txt) | Mapa separado do que já está documentado e do que foi encontrado no onboarding/backend. |
| [`/services-catalog.json`](/services-catalog.json) | Catálogo estruturado em JSON para automações, validadores e agentes. |
| [`/coverage-report.json`](/coverage-report.json) | Relatório estruturado com totais, categorias, pendências e divergências de coverage. |
| [`/examples/*.curl`](/examples/auth.hml.curl) | Chamadas prontas para homologação e produção. |

## Uso recomendado

Para tirar dúvidas rápidas sobre a estrutura da documentação, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms.txt
```

Para pedir ajuda de implementação, geração de `curl`, escolha de service ou
explicação de fluxo, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-small.txt
```

Para gerar exemplos de request a partir dos services documentados, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-api-reference.txt
```

Para análises mais completas, revisão de cobertura ou perguntas que dependem de
todo o conteúdo da documentação, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-full.txt
```

Para comparar o que já está na documentação com o que foi encontrado no
onboarding/backend, use:

```txt
https://react-it.github.io/api-docs-idcerberus/llms-services-coverage.txt
```

Para automações que precisam de dados estruturados, use:

```txt
https://react-it.github.io/api-docs-idcerberus/services-catalog.json
https://react-it.github.io/api-docs-idcerberus/coverage-report.json
```

Para importar exemplos prontos em ferramentas ou scripts, use:

```txt
https://react-it.github.io/api-docs-idcerberus/examples/auth.hml.curl
https://react-it.github.io/api-docs-idcerberus/examples/service-api-cpf.hml.curl
```

## Exemplo de prompt

```txt
Use esta documentação como fonte principal:
https://react-it.github.io/api-docs-idcerberus/llms-small.txt

Preciso integrar a API idCerberus para consultar dados de CPF.
Explique o fluxo, gere o curl de autenticação e depois o curl do service correto.
Não invente services que não estejam na documentação.
```

## Boas práticas para IA

- Use homologação para exemplos e testes.
- Use produção somente quando o usuário pedir explicitamente.
- Nunca exponha `client`, `secret`, token JWT real, CPF real, CNPJ real ou imagem real.
- Se um service não estiver no catálogo, confirme no backend/onboarding antes de documentar.
- Para consultas externas, prefira sempre o endpoint `POST /api/service-api` com o campo `service`.

## Atualização dos arquivos

Os arquivos são gerados a partir dos guias locais, do OpenAPI e do mapeamento de
cobertura dos services. Para atualizar localmente:

```bash
node scripts/generate-llms.mjs
```

No Windows, se o PowerShell bloquear `npm.ps1`, use:

```powershell
npm.cmd run generate:llms
```

Para rodar a verificação de texto e regenerar os arquivos:

```bash
npm run check
```

O export da documentação também executa a geração antes de publicar os arquivos.
