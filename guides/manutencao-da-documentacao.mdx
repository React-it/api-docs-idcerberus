# Manutenção da documentação

Use esta página quando for alterar OpenAPI, guias, exemplos, catálogo de services
ou arquivos usados por LLM/MCP.

## Regra principal

Não edite artefatos gerados como fonte principal. A fonte deve ser o OpenAPI,
os guias em MDX e o script `scripts/generate-llms.mjs`.

Artefatos gerados:

- `llms.txt`
- `llms-small.txt`
- `llms-full.txt`
- `llms-api-reference.txt`
- `services-catalog.json`
- `services-catalog.min.json`
- `mcp-manifest.json`
- `examples/*.curl`
- páginas geradas do catálogo no API Reference

## Fluxo recomendado

1. Altere a fonte correta: guia, OpenAPI ou gerador.
2. Rode `npm run generate:artifacts`.
3. Rode `npm run check`.
4. Revise o diff.
5. Confirme que não entrou arquivo local, credencial, base64, CPF, CNPJ ou token sensível.

## Checks disponíveis

| Comando | O que valida |
| --- | --- |
| `npm run generate:artifacts` | Regenera LLMS, catálogo, manifesto, exemplos e páginas derivadas. |
| `npm run check:text` | Procura texto quebrado, mojibake, placeholders inseguros e links internos quebrados. |
| `npm run check:artifacts` | Valida estrutura do catálogo, manifesto MCP e arquivos obrigatórios. |
| `npm run check:drift` | Regenera artefatos e falha se ficar diff não commitado. |
| `npm run check` | Roda geração, validação de texto e validação de artefatos. |

## Antes de abrir PR

- Confirme que o `service` documentado usa o alias de chamada correto.
- Confirme que exemplos usam placeholders, não dados reais.
- Confirme que OCR, Face e biometria deixam claro o tipo de imagem esperado.
- Confirme que a resposta pública usa `result`, não `fieldsOutput`.
- Confirme que `mcp-manifest.json` e os catálogos JSON continuam válidos.

## Quando mexer no OpenAPI

Sempre rode geração e checks depois de qualquer alteração em
`api-reference/openapi.json`. O catálogo de services, os arquivos LLM e os curls
dependem desse arquivo.

## Quando mexer no gerador

Depois de alterar `scripts/generate-llms.mjs`, rode:

```bash
npm run check
npm run check:drift
```

Se o gerador criar uma página nova, adicione ela também no `docs.json`.
