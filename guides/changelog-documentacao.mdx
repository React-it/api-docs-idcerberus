# Changelog da documentação

Esta página registra as mudanças relevantes da documentação. Use este histórico
para entender o que foi adicionado, revisado ou reorganizado antes de uma nova
publicação.

## Atualização atual

### API Reference

- Foram incluídos exemplos adicionais de request e response no endpoint
  `POST /api/service-api`.
- A descrição do API Reference foi reforçada para explicar que a rota é a mesma
  e que o produto muda pelo campo `service`.
- Foram adicionados termos de busca para facilitar encontrar serviços como CPF
  Receita, CNPJ Receita, KYC, OCR, FaceMatch, Liveness, débitos, protestos e
  dados eleitorais.

### Serviços confirmados via API

- O mapeamento passou a considerar apenas listeners em `events/api` quando o
  objetivo é validar o que está implementado via `POST /api/service-api`.
- O arquivo `llms-api-real.txt` lista os serviços confirmados por código e
  separa esses itens de CSV antigo, SDK, backoffice ou fluxo de onboarding.
- A cobertura atual está fechada em `71` serviços implementados via API e `71`
  serviços documentados.

### Guias

- O Quickstart foi detalhado para Postman, Windows PowerShell, CMD, macOS e
  Linux.
- A página de Status e erros ganhou exemplos práticos de token inválido,
  `service` ausente, documento obrigatório ausente, produto sem permissão e
  fonte externa indisponível.
- Os exemplos por linguagem foram ampliados com `fetch`, Axios, Python, C# e
  Java.

### LLMs

- Os arquivos `llms.txt`, `llms-small.txt`, `llms-full.txt`,
  `llms-api-reference.txt` e `llms-services-coverage.txt` foram regenerados.
- Foi criado o `llms-api-real.txt`, focado apenas em services confirmados via
  API.
- O `services-catalog.json` passou a incluir mais metadados operacionais para
  automações e assistentes.

## Como atualizar este changelog

Ao publicar uma nova versão da documentação, registre:

1. páginas criadas ou removidas;
2. services adicionados ao API Reference;
3. mudanças de exemplos;
4. mudanças em scripts de geração ou validação;
5. resultado da cobertura de services;
6. resultado do checklist de release.
