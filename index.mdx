## Caminho mais curto

<Steps>
  <Step title="Gere um token">
    Use `POST /api/token-generate` com `client` e `secret` do produto.
  </Step>
  <Step title="Escolha o service">
    Abra o [índice de services](/guides/indice-de-services) ou a [API Reference](/api-reference/services-por-caso-de-uso) e copie o alias correto.
  </Step>
  <Step title="Monte o payload">
    Envie `service` e os campos obrigatórios do produto, como `cpf`, `cnpj`, `image1`, `image2`, `documentType` ou `key`.
  </Step>
  <Step title="Execute em HML">
    Chame `POST https://backoffice-hml.idcerberus.com/api/service-api` com `Authorization: Bearer {jwt_token}`.
  </Step>
  <Step title="Leia o resultado">
    Use `result` como retorno de negócio e `status.message`, `onboardingStatus` e `externalId` para diagnóstico.
  </Step>
</Steps>

## Modelos de integração

| Modelo | Quando usar | O que você implementa |
| --- | --- | --- |
| API direta | Consultas server-to-server por CPF, CNPJ, imagem ou chave | Token, payload, chamada HTTP e leitura do `result` |
| SDK | Captura guiada de documento, selfie e dados no aplicativo | Geração do `tokenOnboarding` e consulta do resultado |
| Cliente Web | Fluxos em WebView, navegador ou QR Code | Redirecionamento do usuário e leitura do processamento |

## Endpoints essenciais

| Operação | Endpoint | Uso |
| --- | --- | --- |
| Gerar token | `POST /api/token-generate` | Autenticar chamadas protegidas |
| Executar service | `POST /api/service-api` | Rodar CPF, CNPJ, OCR, biometria, risco e compliance |
| Gerar TokenOnboarding | `POST /api/token-history-onboarding` | Iniciar fluxo via SDK |
| Consultar onboarding | `GET /api/onboarding/report/{tokenOnboarding}` | Obter status, campos e regras processadas |
| Baixar relatório PDF | `GET /api/history-onboarding-report/{tokenOnboarding}` | Armazenar ou auditar o resultado final |

## Exemplo rápido

Gere um token:

```bash
curl --location 'https://backoffice-hml.idcerberus.com/api/token-generate' \
  --header 'Content-Type: application/json' \
  --data '{
    "client": "{client}",
    "secret": "{secret}"
  }'
```

Use o token para executar uma consulta:

```bash
curl --location 'https://backoffice-hml.idcerberus.com/api/service-api' \
  --header 'Authorization: Bearer {jwt_token}' \
  --header 'Content-Type: application/json' \
  --data '{
    "service": "SERVICE_PERSON_DATA_ENRICHMENT_BIGDATACORP",
    "cpf": "00000000000"
  }'
```

## Padrão de resposta

A maior parte dos services retorna os dados úteis em `result` e o status técnico em `status`.

```json
{
  "result": {},
  "status": {
    "code": 200,
    "message": "Success"
  },
  "onboardingStatus": "APPROVED",
  "externalId": "..."
}
```

<Note>
  O campo `service` define qual produto será executado dentro de `POST /api/service-api`. Os demais campos mudam conforme a consulta escolhida.
</Note>
