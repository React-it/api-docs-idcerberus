# Services de pessoa física

Catálogo explícito dos services de pessoa física disponíveis via API, com campos esperados e exemplos de request.

<Info>
Todos os services abaixo usam o mesmo endpoint: `POST /api/service-api`. O produto executado é definido pelo campo `service` no body da requisição.
</Info>

## Como ler esta referência

- **Nome**: nome funcional do produto.
- **Service**: valor exato que deve ser enviado no campo `service`.
- **Família**: agrupamento por objetivo de uso, como dados cadastrais, risco, jurídico ou biometria.
- **Campos**: parâmetros esperados no body além de `service`.
- **Retorno principal**: resumo dos dados esperados no objeto `result` para aquele service.
- **Exemplos**: cada item traz body JSON, curl de homologação, curl de produção e response resumido.
