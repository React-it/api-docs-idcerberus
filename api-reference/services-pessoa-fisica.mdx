Abra um service para ver quando usar, campos obrigatórios, body, curl e response resumido.

<AccordionGroup>
<Accordion title="Busca de face na base">

**Service:** `SERVICE_FACE_INDEX`

**Quando usar:** Use para comparar duas imagens faciais e retornar a similaridade entre elas.

**O que retorna:** Busca uma selfie na base de faces indexadas e retorna se encontrou face, CPF associado e similaridade quando disponíveis.

<CardGroup cols={2}>
 <Card title="Entrada">
 Campos obrigatórios: `service`, `image1`.
 </Card>
 <Card title="Resultado">
 Principais campos em `result`: `cpf`, `faceFound`, `similarity`.
 </Card>
 <Card title="Status da chamada">
 Use `status.code` e `status.message` para entender se a consulta processou corretamente.
 </Card>
 <Card title="Campos opcionais">
 Nenhum campo opcional mapeado neste exemplo.
 </Card>
</CardGroup>

**Endpoint:** `POST /api/service-api`

**Campos obrigatórios:** `service`, `image1`

**Campos opcionais:** Nenhum campo opcional mapeado neste exemplo.

### Passo a passo

1. Gere o token em `POST /api/token-generate` e envie no header `Authorization: Bearer {jwt_token}`.
2. Monte o body com o `service` exato e os campos obrigatórios listados abaixo.
3. Execute `POST /api/service-api` no ambiente escolhido.
4. Confira `status.code` e `status.message` para validar o processamento técnico.
5. Mapeie os campos de `result` conforme o resumo e o exemplo de response deste service.

### Body

```json
{
  "service": "SERVICE_FACE_INDEX",
  "image1": "base64"
}
```

### Homologação

```bash
curl --location 'https://backoffice-hml.idcerberus.com/api/service-api' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {jwt_token}' \
--data '{
  "service": "SERVICE_FACE_INDEX",
  "image1": "base64"
}'
```

### Produção

```bash
curl --location 'https://backoffice.idcerberus.com/api/service-api' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {jwt_token}' \
--data '{
  "service": "SERVICE_FACE_INDEX",
  "image1": "base64"
}'
```

### Campos do body

| Campo | Obrigatório | Descrição |
| --- | --- | --- |
| `service` | Sim | Código exato do produto que será executado pelo endpoint central. |
| `image1` | Sim | Primeira imagem enviada em base64 ou referência equivalente, conforme o produto. |

### Campos principais do result

| Campo | Descrição |
| --- | --- |
| `result.cpf` | CPF relacionado ao resultado da consulta. |
| `result.faceFound` | Indica se a busca facial encontrou uma face correspondente na base. |
| `result.similarity` | Percentual de similaridade retornado em validações biométricas ou faciais. |

### Como consumir o retorno

<CardGroup cols={2}>
 <Card title="result">
 Dados públicos do service. É o objeto principal para mapear no sistema do cliente.
 </Card>
 <Card title="status">
 Status técnico da chamada, com `code` e `message`.
 </Card>
 <Card title="onboardingStatus">
 Quando retornado, resume o desfecho operacional: `APPROVED`, `REFUSED` ou `ERROR`.
 </Card>
 <Card title="externalId">
 Identificador para rastrear a consulta em suporte, logs ou auditoria.
 </Card>
</CardGroup>

### Response resumido

```json
{
  "result": {
    "cpf": "cpf",
    "faceFound": true,
    "similarity": 98.42
  },
  "status": {
    "code": 200,
    "message": "Success"
  },
  "externalId": "{externalId}"
}
```

Neste service, o objeto `result` representa: Busca uma selfie na base de faces indexadas e retorna se encontrou face, CPF associado e similaridade quando disponíveis.
